# CN Code 会话、记忆与指令加载规格说明

## 1. 背景与目标

CN Code 已经具备 Web Chat、TUI、Agent Loop、工具调用、权限检查、MCP 接入、系统提示词组装和上下文压缩能力。随着项目使用时间变长，用户会希望 CN Code 能在不同启动之间保留关键上下文，例如项目约定、用户偏好、上次会话的重要决策，以及特定项目里的指令文件。

本章目标是补齐一套轻量的会话与记忆基础设施：

1. 按优先级发现并加载项目指令文件，把稳定规则注入模型上下文。
2. 支持指令文件里的 `@include` 模块化引用，让项目规则可以拆分维护。
3. 用 JSONL 格式持久化会话，支持会话恢复，并能校验消息链完整性。
4. 在 Agent 对话结束后自动抽取值得记住的信息，写入本地 `memories.md`。
5. 下次会话启动时自动加载记忆，并注入系统提示或消息上下文。
6. 提供 Web UI 优先的 `/session` 和 `/memory` 命令，用于管理会话和记忆。

本章不做向量数据库、RAG 检索和语义相关性排序。

## 2. 功能需求

### F1. 指令文件发现

系统必须能在当前项目中按固定优先级发现指令文件。

当存在多个候选文件时，系统只加载优先级最高的一组入口，并把其内容作为项目级稳定规则注入模型上下文。

### F2. 指令文件 include

指令文件必须支持 `@include` 形式的模块化引用。

被引用文件必须位于项目工作目录内。系统加载入口指令时，应递归展开 include 内容，并避免循环引用导致无限递归。

### F3. 指令注入

加载后的指令内容必须进入最终模型请求上下文。

项目指令应作为稳定上下文参与系统提示词组装，优先级高于普通对话消息，但低于 CN Code 内置安全和行为规则。

### F4. 会话持久化

系统必须把会话消息以 JSONL 格式保存到当前项目下的会话目录。

每条记录必须能表达消息角色、消息内容、时间戳和完整性校验信息。保存失败不得破坏正在进行的对话。

### F5. 会话恢复

系统必须能列出当前项目的历史会话，并恢复指定会话到当前对话历史。

恢复时必须校验消息链完整性。校验失败时，系统必须拒绝静默恢复损坏历史，并给出可见提示。

### F6. 自动记忆抽取

Agent 对话结束后，系统应在后台从本轮对话中抽取值得长期保留的信息。

记忆抽取应关注用户偏好、项目约定、重要决策、长期任务线索和用户明确要求记住的内容。抽取结果写入当前项目的 `memories.md`。

### F7. 记忆加载与注入

新会话启动时，系统必须自动读取当前项目的 `memories.md`。

当记忆非空时，系统必须把记忆注入模型上下文，让 Agent 在后续回答中能参考这些长期信息。

### F8. 记忆管理命令

用户必须能通过 Web Chat 输入 `/memory` 管理记忆。

命令至少支持查看当前记忆、追加记忆、清空记忆。命令不得作为普通用户消息发送给模型。

### F9. 会话管理命令

用户必须能通过 Web Chat 输入 `/session` 管理会话。

命令至少支持查看当前会话信息、列出历史会话、恢复指定会话。命令不得作为普通用户消息发送给模型。

### F10. Web UI 反馈

会话和记忆命令执行后，Web UI 必须显示可见的系统消息。

当命令失败、会话损坏、include 文件不存在或 include 越界时，系统必须给出清晰提示，并保持当前对话状态不被破坏。

## 3. 非功能要求

### N1. 崩溃安全

会话持久化必须尽量避免写入半条记录。单次写入失败时，不得清空已有会话文件。

### N2. 完整性校验

会话恢复必须能发现消息缺失、顺序错乱或内容被修改的情况。

完整性校验用于拒绝损坏恢复，不用于安全加密或防止恶意篡改。

### N3. 路径安全

指令 include、会话文件和记忆文件都必须限制在当前项目或 CN Code 固定数据目录内。

模型不能通过 include 或命令参数读取工作目录外的任意文件。

### N4. 后台执行

自动记忆抽取不能阻塞 Web UI、TUI 或 Agent Loop 主流程。

抽取失败时只记录或显示可见提示，不影响用户继续对话。

### N5. 人类可读

会话文件使用 JSONL，便于追加、排查和部分恢复。

记忆文件使用 Markdown，便于用户直接阅读和手工编辑。

### N6. Web UI 优先

本章的命令交互以 Web Chat 为主。REPL 和 TUI 可复用底层能力，但不要求在本章补齐同等命令体验。

### N7. 与现有提示词兼容

新加载的指令和记忆必须接入现有提示词组装流程，不得绕过内置身份、安全、工具使用和上下文管理规则。

## 4. 设计骨架

本章拆成四个能力边界：

1. 指令加载器：负责发现项目指令文件、展开 `@include`、做路径约束与循环检测，并输出可注入的项目指令文本。
2. 会话存储：负责按 JSONL 追加保存消息、列出历史会话、恢复会话，并做消息链完整性校验。
3. 记忆管理：负责读取和写入 `memories.md`，在对话结束后后台抽取长期记忆，并在新会话启动时提供记忆内容。
4. Web 命令入口：负责拦截 `/session` 和 `/memory`，调用底层管理能力，把结果作为系统消息返回给 Web UI。

主接入点包括：

1. 应用启动或 Agent Loop 构造时加载项目指令和记忆，并交给现有提示词组装模块。
2. Agent Loop 每轮开始或结束时触发会话消息落盘。
3. Agent Loop 完成一轮用户任务后触发后台记忆抽取。
4. Web Chat 的命令处理分支拦截 `/session` 和 `/memory`。

## 5. Out of Scope

1. 不实现向量数据库。
2. 不实现 RAG 检索。
3. 不实现语义相关性排序或记忆召回评分。
4. 不实现跨项目共享记忆。
5. 不实现远程同步会话。
6. 不实现会话加密。
7. 不实现复杂命令参数解析器。
8. 不实现记忆去重、过期和自动合并。
9. 不要求 REPL/TUI 拥有与 Web Chat 完全一致的命令界面。

## 6. 完成定义

当根目录 `checklist.md` 中所有可观测验收项通过，并且相关自动化测试通过后，本章视为完成。
