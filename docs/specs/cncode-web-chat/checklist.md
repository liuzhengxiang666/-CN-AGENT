# CN Code Web 聊天界面验收清单

## 1. 启动与端口

- [ ] 执行 `cncode --web` 后，本地 HTTP 服务启动。
- [ ] 服务监听 `127.0.0.1`。
- [ ] 默认优先使用端口 `8765`。
- [ ] 当 `8765` 被占用时，自动使用后续可用端口。
- [ ] 终端打印实际访问 URL。
- [ ] 服务启动后尝试自动打开浏览器。
- [ ] 自动打开浏览器失败时，服务仍保持运行。

## 2. 页面布局

- [ ] 页面包含聊天区。
- [ ] 页面包含输入区。
- [ ] 页面包含侧边栏。
- [ ] 侧边栏显示 CN Code 标题。
- [ ] 侧边栏显示当前 `protocol/model`。
- [ ] 侧边栏显示服务连接状态。
- [ ] 侧边栏显示当前消息数量。
- [ ] 侧边栏显示 `Tools: not enabled`。

## 3. 输入行为

- [ ] 在输入框输入一行文本后按 Enter，消息被发送。
- [ ] 发送后输入框清空。
- [ ] 在输入框按 Shift+Enter，插入换行。
- [ ] Shift+Enter 不发送消息。
- [ ] 点击发送按钮也能发送消息。
- [ ] 空消息不会发送。

## 4. 流式回复

- [ ] 发送真实问题后，页面立即显示用户消息。
- [ ] 页面创建助手消息占位。
- [ ] 模型回复片段通过 SSE 逐步显示。
- [ ] 回复不是等待完整生成后才显示。
- [ ] 回复过程中页面自动滚动到底部。
- [ ] 回复完成后可继续发送下一条消息。

## 5. 多轮上下文

- [ ] 发送：“我叫小明，喜欢 Java。”
- [ ] 等待模型回复完成。
- [ ] 继续发送：“我刚才说我喜欢什么语言？”
- [ ] 模型回复能参考前文，并指出用户喜欢 Java。
- [ ] 后端进程退出后，不要求保留历史。

## 6. 错误与安全

- [ ] Provider 调用失败时，页面显示错误提示。
- [ ] 错误后用户仍可继续发送消息。
- [ ] 忙碌时重复发送会被拒绝或提示。
- [ ] `/api/status` 不返回 API Key。
- [ ] HTML/CSS/JS 不包含 API Key。
- [ ] 服务不提供任意文件读取接口。

## 7. 模式回归

- [ ] 不带参数启动时，现有默认模式仍可用。
- [ ] `--repl` 仍进入基础 REPL。
- [ ] `--tui` 仍尝试进入 TUI。
- [ ] Web 模式不删除现有 REPL/TUI 代码。

## 8. 构建验证

- [ ] `.\gradlew.bat check` 通过。
- [ ] `.\gradlew.bat installDist` 通过。
- [ ] 生成的 `build/install/cncode/bin/cncode.bat --web` 可启动。
