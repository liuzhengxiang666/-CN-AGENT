# CN Code TUI 对话界面验收清单

## 1. 启动与布局

- [ ] 执行 `cncode` 后优先进入 TUI。
- [ ] TUI 显示历史对话区域。
- [ ] TUI 显示多行输入框。
- [ ] TUI 显示底部状态栏。
- [ ] 状态栏包含 `default`。
- [ ] 状态栏包含当前 `protocol/model`，例如 `openai/deepseek-v4-flash`。
- [ ] 终端窗口较小时不会直接崩溃；无法启动 TUI 时进入 REPL fallback。

## 2. 输入行为

- [ ] 用户输入普通字符后，内容显示在输入框中。
- [ ] 按 Enter 会在输入框中插入换行。
- [ ] 按 Ctrl+Enter 会发送当前输入。
- [ ] 发送后输入框清空。
- [ ] 输入完整内容 `exit` 并发送后，程序正常退出。
- [ ] 输入完整内容 `quit` 并发送后，程序正常退出。
- [ ] 模型正在回复时再次发送，不会破坏界面状态。

## 3. 流式回复

- [ ] 发送真实问题后，CN Code 调用当前配置的 Provider。
- [ ] 回复内容逐字或逐块显示在历史对话区域。
- [ ] 回复不是等待完整生成后才出现。
- [ ] 流式输出过程中界面持续刷新。
- [ ] 回复完成后，助手完整消息保存在当前会话历史。

## 4. 多轮上下文

- [ ] 发送：“我叫小明，喜欢 Java。”
- [ ] 等待模型回复完成。
- [ ] 继续发送：“我刚才说我喜欢什么语言？”
- [ ] 模型回复能参考前文，并指出用户喜欢 Java。
- [ ] 退出程序后再次启动，不要求保留上次历史。

## 5. 历史滚动

- [ ] 发送多轮消息，使历史内容超过可视区域高度。
- [ ] 鼠标滚轮向上可以查看较早消息。
- [ ] 鼠标滚轮向下可以回到较新消息。
- [ ] 模型流式回复时，历史区域自动滚动到底部。
- [ ] 鼠标滚轮事件不会导致程序异常退出。

## 6. 错误与 fallback

- [ ] Provider 调用失败时，错误信息显示在历史对话区域。
- [ ] Provider 调用失败后，用户仍可继续输入。
- [ ] TUI 初始化失败时，程序打印中文提示。
- [ ] TUI 初始化失败后，程序进入基础 REPL。
- [ ] REPL fallback 仍支持流式回复、多轮上下文和退出命令。

## 7. 构建与回归

- [ ] `.\gradlew.bat check` 通过。
- [ ] `.\gradlew.bat installDist` 通过。
- [ ] 生成的 `build/install/cncode/bin/cncode.bat` 可启动。
- [ ] 现有配置读取、Provider 工厂、会话模型、REPL 冒烟测试仍通过。
- [ ] 项目继续使用 Java 21。
