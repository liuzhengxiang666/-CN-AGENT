# CN Code 上下文压缩验收清单

## 工具结果轻量预防

- [ ] 存在 `src/main/java/cncode/toolresult/ContentReplacementState.java`。
- [ ] `ContentReplacementState` 能记录已见工具调用 ID。
- [ ] `ContentReplacementState.copy()` 生成独立副本，修改副本不影响原对象。
- [ ] 存在 `ContentReplacementRecord`，字段包含 `kind`、`toolUseId`、`replacement`。
- [ ] 存在 `ReplacementRecordsIO.append(...)`。
- [ ] 存在 `ReplacementRecordsIO.load(...)`。
- [ ] 缺失 replacement records 文件时，load 返回空列表且不抛异常。
- [ ] 单个工具结果超过阈值时，完整内容写入会话目录文件。
- [ ] 超限工具结果在 API 对话视图里被替换成预览文本。
- [ ] 预览文本包含原始长度。
- [ ] 预览文本包含可读的文件路径。
- [ ] 预览文本提示需要细节时重新读取文件。
- [ ] 同一个工具结果重复处理时，替换文本保持一致。
- [ ] 同一条消息内工具结果合计超限时，系统优先替换较大的结果。
- [ ] 工具结果替换后，工具结果顺序不变。
- [ ] `ToolResultBudget.apply(...)` 不修改原始 `ConversationManager`。

## 整体上下文压缩

- [ ] 存在 `src/main/java/cncode/compact/ContextCompactor.java`。
- [ ] 存在 token 估算方法，覆盖普通文本、工具调用参数、工具结果和 thinking。
- [ ] 空对话 token 估算不会抛异常。
- [ ] 自动压缩在低于阈值时不触发。
- [ ] 自动压缩在达到阈值时触发摘要请求。
- [ ] 摘要请求传空工具列表或等价禁用工具参数。
- [ ] 摘要 Prompt 开头包含禁止调用工具的说明。
- [ ] 摘要 Prompt 结尾再次包含禁止调用工具的说明。
- [ ] 摘要 Prompt 要求输出分析草稿和正式摘要。
- [ ] 系统只把正式摘要写回 conversation。
- [ ] 正式摘要包含“主要请求”部分。
- [ ] 正式摘要包含“关键概念”部分。
- [ ] 正式摘要包含“文件和代码”部分。
- [ ] 正式摘要包含“错误与修复”部分。
- [ ] 正式摘要包含“解决过程”部分。
- [ ] 正式摘要包含“用户原话”部分。
- [ ] 正式摘要包含“待办事项”部分。
- [ ] 正式摘要包含“当前工作状态”部分。
- [ ] 正式摘要包含“下一步”部分。
- [ ] 压缩后 conversation 包含边界消息。
- [ ] 边界消息提示需要文件细节时重新读取。
- [ ] 边界消息提示不要根据摘要脑补代码。

## 熔断与失败处理

- [ ] 存在自动压缩连续失败计数。
- [ ] 自动压缩失败后不破坏原 conversation。
- [ ] 自动压缩连续失败达到上限后停止自动触发。
- [ ] 自动压缩成功后失败计数清零。
- [ ] 手动压缩不受自动熔断限制。
- [ ] 手动压缩失败时向用户显示清楚错误。

## Agent Loop 接入

- [ ] 每次 Provider 请求前先执行工具结果轻量预防。
- [ ] 每次 Provider 请求前再执行整体上下文兜底压缩。
- [ ] Provider 实际收到的是压缩后的 API 对话视图。
- [ ] 最新用户请求不会被整体摘要替换。
- [ ] 压缩成功时上层能收到 compact 事件。
- [ ] `/compact` 不进入普通对话历史。
- [ ] `/compact` 能手动触发整体压缩。
- [ ] Web UI 能展示压缩完成提示。

## 端到端验收

- [ ] 构造一个返回超长内容的工具调用后，`.cncode` 或会话目录下出现工具结果落盘文件。
- [ ] 对话历史中对应工具结果变成预览和路径，而不是完整超长内容。
- [ ] 让模型继续处理该结果时，它能通过读文件工具重新读取落盘文件。
- [ ] 构造长对话触发自动压缩后，Web UI 出现压缩提示。
- [ ] 自动压缩后继续提问，模型仍能基于摘要和最近消息回答。
- [ ] 连续制造摘要失败，达到上限后自动压缩不再反复触发。
- [ ] 执行 `.\gradlew.bat check --console=plain` 通过。
