# CN Code TUI UI 优化规格说明

## 1. 背景与目标

当前 CN Code 已有 Lanterna TUI 对话界面，但视觉效果较基础，输入行为也与用户期望不一致。本阶段目标是在不改动 Provider、配置、会话和工具相关架构的前提下，优化 TUI 交互与视觉表现。

优化后，用户在 TUI 中按 Enter 直接发送当前输入；按 Ctrl+Enter 在输入框中插入换行。历史对话区域和输入区域的文本整体支持左右位置微调，使用户可以通过快捷键调整界面文本的横向位置。

## 2. 功能需求

### F1. Enter 发送

用户在输入框中按 Enter 时，发送当前输入内容。

发送成功后输入框清空。

### F2. Ctrl+Enter 换行

用户在输入框中按 Ctrl+Enter 时，在当前输入位置插入换行。

### F3. 文本横向位置调整

用户可以通过快捷键调整 TUI 文本整体横向偏移。

- `Ctrl+Left`：文本整体向左移动。
- `Ctrl+Right`：文本整体向右移动。

偏移应影响历史对话区域和输入区域。

### F4. 偏移边界

文本偏移不得小于 0。

文本偏移不得大到导致主要内容完全不可见。

### F5. 视觉优化

TUI 应比当前版本更精致，至少包括：

- 更清晰的顶部标题或欢迎信息。
- 历史对话区域、输入区域和状态栏边界更明确。
- 用户消息、助手消息、系统消息视觉区分更清晰。
- 输入框提示更明确。
- 状态栏继续显示 `default    protocol/model`。

## 3. 非功能需求

### N1. 不改 Provider

本阶段不修改 OpenAI、Anthropic、DeepSeek 相关 Provider 行为。

### N2. 不改配置格式

本阶段不修改 YAML 配置格式。

### N3. 保持 fallback

TUI 启动失败或非交互环境下，仍 fallback 到基础 REPL。

### N4. 保持 Java 21 和 Lanterna

继续使用 Java 21 和 Lanterna。

## 4. 不做的事

1. 不新增 tool calling。
2. 不新增模型切换命令。
3. 不新增 token 统计。
4. 不新增主题配置文件。
5. 不新增鼠标拖拽调整布局。
6. 不改变现有 Provider 抽象。

## 5. 验收标准

### A1. Enter 发送验收

用户输入一行内容后按 Enter，消息被发送，输入框清空。

### A2. Ctrl+Enter 换行验收

用户输入内容时按 Ctrl+Enter，输入框中插入换行，不发送消息。

### A3. 左右偏移验收

用户按 Ctrl+Right 后，历史区域和输入区域文本整体向右移动。

用户按 Ctrl+Left 后，历史区域和输入区域文本整体向左移动。

### A4. 偏移边界验收

连续按 Ctrl+Left 时，文本不会移动到负偏移。

连续按 Ctrl+Right 时，主要内容仍可见。

### A5. 视觉验收

启动 TUI 后，界面比旧版更清晰：有标题、有分隔线、有明确输入提示，消息角色区分明显。

### A6. 回归验收

流式回复、多轮上下文、鼠标滚轮、退出命令和 REPL fallback 仍可用。
